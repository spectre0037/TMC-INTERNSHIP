1. SDLC stages, one sentence each: Requirements — figure out what the software needs to do and for whom. Design — plan architecture before coding. Development — write the software. Testing — check it works before users do. Deployment — release to production. Maintenance — fix and improve it post-launch.
2. Waterfall vs Agile: Waterfall completes each stage fully before starting the next, in strict sequence. Agile repeats short cycles (sprints) of build-test-adjust continuously instead of doing it all once.
3. How AI is changing each stage: Requirements — drafts specs/user stories from conversations. Design — proposes architecture options/diagrams. Development — coding assistants (Copilot/Claude) write and complete code live. Testing — auto-generates test cases including edge cases. Deployment — reviews PRs and flags risky changes before merge. Maintenance — AIOps monitors production and predicts/flags failures automatically.
4. Risks of leaning on AI: hallucinated code (calls to functions/libraries that don't exist), insecure code if unreviewed, over-trust leading developers to stop checking output, and the fact that AI doesn't understand real business intent — a human must still verify.
5. Agentic AI / AI-first development: AI systems that autonomously plan, execute multi-step tasks, and act with minimal human supervision, rather than just responding to single prompts — spanning multiple SDLC stages (plan, code, test, explain) alongside human developers. This mirrors TMC's AiNow idea that AI should be woven into every layer of the business, not added as a single bolted-on tool.

"Where software is heading" (solo, 137 words):
In five years, I don't think the software engineer's job disappears — it moves up a level. Agentic AI is already handling full slices of the SDLC on its own: writing code, running tests, debugging, even opening pull requests without a human at every step. What doesn't change is judgment — deciding what "correct" means for a real business, weighing tradeoffs, and being accountable when something breaks in production. Engineers will spend less time typing code and more time specifying intent, reviewing AI output critically, and catching the subtle mistakes AI doesn't know it's making. That mirrors TMC's own AiNow philosophy: AI isn't replacing the professional, it's becoming the layer underneath everything they do. The job survives — it just moves from "writer of code" to "director and verifier of AI-written code."
